# Request Interceptor — Контекст для копирайтера

## Что это за продукт

**Request Interceptor** — Chrome-расширение для разработчиков и QA-инженеров. Позволяет перехватывать HTTP/WebSocket/GraphQL-запросы в браузере и подменять ответы на лету, без зависимости от реального бэкенда.

**Одним предложением:** Мокаешь API прямо в браузере — никакого сервера, никаких прокси, просто правила и ответы.

---

## Целевая аудитория

- **Frontend-разработчики** — тестируют UI без ожидания бэкенда
- **QA-инженеры** — симулируют ошибки, задержки, edge-case данные
- **Fullstack-разработчики** — отлаживают фронт, когда бэкенд ещё не готов или недоступен

**Боль:** При тестировании UI нужно симулировать разные ответы API — ошибки, пустые данные, тайм-ауты. Сейчас для этого либо поднимают mock-сервер, либо хардкодят данные в коде, либо используют громоздкие инструменты. Request Interceptor решает это одной кнопкой в браузере.

## Ключевые фичи

### Бесплатные (Free)

| Фича | Описание |
|-------|----------|
| **Mock Rules** | Создание правил подмены: URL-паттерн (exact/wildcard/regex) + HTTP-метод + тело → кастомный ответ (JSON, raw text) |
| **Статус-коды и заголовки** | Любой HTTP-статус (200, 404, 500...) + произвольные заголовки ответа |
| **Задержки** | Настраиваемый delay (мс) для симуляции медленного API |
| **Toggle** | Быстрое включение/выключение отдельных правил |
| **WebSocket** | Перехват WebSocket-соединений, мок сообщений |
| **GraphQL** | Перехват GraphQL-запросов, матчинг по имени операции |
| **Коллекции** | Группировка правил в именованные коллекции |
| **Импорт/Экспорт** | JSON-файл с коллекциями — поделись с коллегами |
| **Запись (Record)** | Запись реальных ответов API и сохранение как mock-шаблоны |
| **Request Log** | Лог перехваченных запросов в реальном времени (MOCKED/REAL бейджи) |
| **Dark/Light mode** | Тёмная тема по умолчанию, как любят разработчики |

## Как это работает (технически, простым языком)

1. Пользователь устанавливает расширение из Chrome Web Store
2. Открывает Side Panel (боковая панель Chrome) — это основной интерфейс
3. Создаёт mock-правило: "Когда GET-запрос на `*/api/users*`, верни вот этот JSON со статусом 200"
4. Расширение перехватывает fetch/XHR-запросы на нужных вкладках и подменяет ответ
5. Разработчик видит мокнутый ответ прямо на странице — без перезагрузки, без прокси

**Side Panel** — основной рабочий интерфейс. Открывается сбоку в Chrome и остаётся открытым, пока пользователь работает с сайтом. Не мешает, всегда под рукой.

## Конкуренты

| Продукт | Отличие от Request Interceptor |
|---------|-------------------------------|
| **Requestly** | Тяжёлый, много функций помимо моков, complex UI |
| **Mocky.io** | Веб-сервис (не расширение), нужен интернет, нет перехвата |
| **MSW (Mock Service Worker)** | Библиотека в коде (не UI), требует настройки в проекте |
| **Postman Mock Server** | Серверный mock, не в браузере, привязка к Postman |
| **Charles / Fiddler** | Десктопные прокси — тяжёлые, сложная настройка |

**Наше преимущество:** Легкое расширение прямо в Chrome, нулевая настройка, визуальный редактор, работает с WebSocket/GraphQL, командные фичи.
